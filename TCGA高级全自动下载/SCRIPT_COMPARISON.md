## 📋 功能详细对比

### 1. 数据下载

**tcga_complete_analysis.R:**
```r
- 自动保存下载进度
- 失败自动重试
- 支持断点续传
- 错误详细记录
```

**tcga_analysis.R:**
```r
- 一次性下载
- 失败需重新开始
- 无进度保存
```

### 2. 差异分析

**tcga_complete_analysis.R:**
```r
- 优先使用DESeq2
- 失败自动切换limma
- 保存完整结果对象
- 多种输出格式
```

**tcga_analysis.R:**
```r
- 默认使用DESeq2
- 失败直接报错
- 保存基本结果
```

### 3. 可视化

**tcga_complete_analysis.R:**
```r
- SCI期刊风格
- 统一配色方案
- 自动调整参数
- 高分辨率输出
```

**tcga_analysis.R:**
```r
- 基础图表
- 标准配色
- 固定参数
```

### 4. 错误处理

**tcga_complete_analysis.R:**
```r
- 每步try-catch包装
- 错误详细记录
- 自动降级策略
- 不中断流程
```

**tcga_analysis.R:**
```r
- 基础错误处理
- 错误直接中断
- 需手动修复
```

## 🔄 从基础脚本迁移到完整脚本

如果你已经用 tcga_analysis.R 做了部分分析，可以这样迁移：

### 步骤1: 备份数据
```r
# 复制你的工作目录
dir.create("backup_analysis")
file.copy(".", "backup_analysis", recursive = TRUE)
```

### 步骤2: 保留已下载数据
```r
# tcga_complete_analysis.R 会自动检测 GDCdata/ 目录
# 如果数据已下载，会直接使用，不会重复下载
```

### 步骤3: 运行新脚本
```r
source("tcga_complete_analysis.R")
```

### 步骤4: 对比结果
```r
# 对比差异基因数量
old_degs <- read.csv("backup_analysis/DESeq2_results_significant.csv")
new_degs <- read.csv("results/tables/differential_expression_significant.csv")

print(paste("旧方法:", nrow(old_degs), "个差异基因"))
print(paste("新方法:", nrow(new_degs), "个差异基因"))
```

## 🔍 常见问题

### Q: 我应该用哪个脚本？
**A**: 如果你不确定，用 tcga_complete_analysis.R。它更稳定可靠。

### Q: 两个脚本的结果一样吗？
**A**: 核心分析逻辑相同，结果应该一致。差异可能来自：
- 随机种子设置
- 过滤参数
- 归一化方法选择

### Q: 可以混用两个脚本吗？
**A**: 可以，但建议：
- 在同一个项目中选择一个
- 数据可以共享（GDCdata/目录）
- 结果分开保存

### Q: 哪个脚本更适合发表论文？
**A**: tcga_complete_analysis.R
- 完整的日志记录
- 可重复的分析流程
- 标准化的输出
